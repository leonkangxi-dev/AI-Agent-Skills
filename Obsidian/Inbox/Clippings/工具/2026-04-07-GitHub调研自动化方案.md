# GitHub调研自动化方案

## 来源
暗夜匿影系统设计

## 核心摘要
当用户分享GitHub链接时，自动获取仓库信息并生成调研报告，同步到Obsidian

## 触发条件
用户分享 github.com/{owner}/{repo} 链接

## 执行流程

### 1. URL解析
```
github.com/user/repo → {owner: user, repo: repo}
```

### 2. 获取仓库信息
- 通过 GitHub API 获取: stars, forks, description, language
- 读取 README.md 内容

### 3. 本地测试 (可选)
- Clone 仓库到本地 (~/github-research/{owner}/{repo})
- 尝试运行/部署

### 4. 生成调研报告
路径: Inbox/github-research/{owner}-{repo}.md

## 技术实现

### GitHub API调用
```python
import requests

def get_repo_info(owner, repo):
    url = f"https://api.github.com/repos/{owner}/{repo}"
    headers = {"Accept": "application/vnd.github.v3+json"}
    return requests.get(url, headers=headers).json()
```

## 注意事项

1. **API限制**: 未认证请求每小时60次，可配置GitHub Token提升
2. **隐私**: 不Clone私有仓库
3. **安全**: 克隆前检查仓库评分(可用skill-vetter)
4. **性能**: 大仓库只获取必要信息
