# 大司命知识体系架构

> 基于 LLM Wiki 模式构建的个人知识库

---

## 核心理念

**AI 协作者 + 人类策划者**

- 你负责寻找、探索、提出好问题
- 我负责总结、交叉引用、归档和维护

## 命名规范

### 文件命名

- **页面**: `主题-子主题.md` (中横线连接)
  - `AI-LLM-提示工程.md`
  - `投资-ETF配置策略.md`

- **元数据**: YAML Frontmatter
```yaml
---
title: 页面标题
type: concept|entity|source|note
tags: [tag1, tag2]
created: 2026-04-15
source: 来源链接
status: draft|active|archived
---
```

## 工作流程

### 1. 摄取 (Ingest)

当你分享链接或文章时：

1. 我读取内容
2. 提取关键信息
3. 归档到对应分类
4. 更新 index.md
5. 记录到 log.md

### 2. 查询 (Query)

你提问时：

1. 我先查 index.md 找相关页面
2. 读取相关内容
3. 综合回答并标注来源
4. 好的回答可归档为新页面

### 3. 整理 (Lint)

定期检查：

- 过时内容 → 更新或归档
- 孤立页面 → 建立链接
- 矛盾信息 → 标记提醒
