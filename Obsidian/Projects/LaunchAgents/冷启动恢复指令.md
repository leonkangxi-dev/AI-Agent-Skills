# 冷启动恢复指令

**创建日期:** 2026-04-21
**用途:** 宿主机重启后的自愈流程

---

## ⚠️ 重要：每次启动必须执行

当暗夜重新加载后，必须第一时间执行以下检查流程：

---

## Step 1: 确认 LaunchAgent 状态

```bash
launchctl list | grep -E "openclaw|nightly"
```

**预期输出:**
```
1513  ai.openclaw.gateway
17272 com.openclaw.nightly
```

---

## Step 2: 验证核心服务端口

```bash
for port in 3000 5000 5001 5002 9996 9997 9998 18789; do
  curl -sf http://localhost:$port >/dev/null 2>&1 && echo "端口 $port: 🟢" || echo "端口 $port: 🔴"
done
```

---

## Step 3: 启动自检守护

```bash
nohup python3 ~/Documents/Obsidian/90_System/health_check.py > ~/logs/health_check.log 2>&1 &
```

---

## Step 4: 验证 Memory 索引

```bash
# 检查 ChromaDB 是否存在
ls -la ~/Documents/Obsidian/90_System/Memory_DB/

# 重新索引（如需要）
~/.openclaw/memory_venv/bin/python ~/.openclaw/workspace/services/memory_indexer.py
```

---

## Step 5: 向执行宫汇报

系统恢复后，自动推送以下格式的信息：

> 执行官，系统已从意外中断中恢复，当前内存占用 [X]%，长期记忆索引已重新挂载。

---

## 快速一键恢复

```bash
# 保存为 restore.sh
#!/bin/bash
launchctl load ~/Library/LaunchAgents/com.openclaw.nightly.plist
sleep 5
~/.openclaw/start_all_services.sh
nohup python3 ~/Documents/Obsidian/90_System/health_check.py > ~/logs/health_check.log 2>&1 &
```

---

## 文件完整性检查

每次启动必须验证以下文件存在：

- [ ] `~/Library/LaunchAgents/com.openclaw.nightly.plist`
- [ ] `~/Documents/Obsidian/90_System/health_check.py`
- [ ] `~/.openclaw/start_all_services.sh`
- [ ] `~/.openclaw/memory_venv/bin/python`
