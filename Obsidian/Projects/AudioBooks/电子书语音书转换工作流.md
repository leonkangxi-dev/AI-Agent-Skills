# 电子书语音书转换工作流

> **功能状态：** 已就绪  
> **最后更新：** 2026-04-23

---

## 功能概述

将电子书（PDF、EPUB、MOBI、AZW3、TXT）自动转化为 MP3 语音书，完成跨平台归档与知识沉淀。

**输入** → **解析** → **切片** → **渲染** → **推送 + 归档**

---

## 支持格式

| 格式      | 解析工具                         | 状态    |
| ------- | ---------------------------- | ----- |
| `.pdf`  | PyMuPDF                      | ✅ 可用  |
| `.epub` | EbookLib                     | ⏳ 需安装 |
| `.mobi` | python-mobi / amazon-killing | ⏳ 需安装 |
| `.azw3` | amazon-killing               | ⏳ 需安装 |
| `.txt`  | 内置文本读取                       | ✅ 可用  |

## 工作流详解

### Step 1 — 文件识别与解析

根据后缀名调用解析工具，提取书名、作者、全书纯文本。

```
.pdf  → PyMuPDF (fitz)  提取文本 + 元数据
.epub → EbookLib        解析章节 + 纯文本
.mobi → mobi            解包 + 文本提取
.azw3 → amazon-killing  解压缩 + 解析
.txt  → 原始文本读取    编码检测后直接读取
```

### Step 2 — 文本预处理

- 去除多余空行和页眉页脚噪声
- 按章节拆分，便于分段渲染
- 切片阈值：单段落超过 3000 字自动截断

### Step 3 — 音频渲染

调用 edge-tts：

```bash
edge-tts \
  --text "文本内容" \
  --voice zh-CN-YunxiNeural \
  --output "书名_chapter_01.mp3"
```

若文本超长，自动分片渲染后合并为完整 MP3。

### Step 4 — Telegram 推送

转换完成后，通过 Bot 发送：

> 📖 书籍《**{书名}**》转换完成，已为您推送。

附件：MP3 文件

### Step 5 — 双重归档

**物理归档：**
```
~/Users/jiang/Desktop/hermes/AudioBooks/
  └── {书名}/
      ├── {书名}_Part1.mp3
      ├── {书名}_Part2.mp3
      └── metadata.json
```

**知识归档（Obsidian）：**
```
~/Documents/Obsidian/Projects/AudioBooks/
  └── {书名}.md
```

笔记模板：

```markdown
# {书名} - 有声化记录

- **转换时间：** {current_date}
- **原始格式：** {format}
- **作者：** {author}
- **本地链接：** [[{mp3_filename}]]
- **状态：** 已同步 Telegram
```

## 故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| `.mobi` 上传被拒 | 网关白名单无此类型 | 已添加至 `SUPPORTED_DOCUMENT_TYPES`（base.py），需重启 Gateway |
| edge-tts 连接失败 | 网络或凭证问题 | 检查 `edge-tts --test` |
| 音频有杂音 | 解析文本含乱码 | 预处理加编码检测（utf-8/gb2312/gbk）|
