## 二、SCADA系统核心架构

### 2.1 SCADA层级结构

```
┌─────────────────────────────────────────┐
│           Level 4 - 企业网络            │
├─────────────────────────────────────────┤
│     Level 3 - 运营管理 (HMI/历史库)      │
├─────────────────────────────────────────┤
│      Level 2 - SCADA主站 (数据采集)       │
├─────────────────────────────────────────┤
│      Level 1 - PLC/RTU (现场控制)        │
└─────────────────────────────────────────┘
```

### 2.2 核心组件

| 组件 | 功能 |
|------|------|
| **PLC/RTU** | 直接连接传感器和执行器，进行实时控制 |
| **SCADA Master** | 数据采集、命令下发、人机界面 |
| **HMI** | 操作员监控界面 |
| **Historian** | 数据存储与分析 |
| **通信网络** | 连接各层级设备 |

### 2.3 常用协议

| 协议 | 类型 | 特点 |
|------|------|------|
| Modbus RTU | 串行 | 开放式，简单可靠 |
| IEC 60870-5-101/104 | 工业标准 | 电力系统常用 |
| IEC 61850 | 以太网 | 变电站自动化 |
| DNP3 | 分布式 | 市政/电力应用 |
| OPC-UA | 现代标准 | 安全、跨平台 |

## 三、双目深度相机技术概述

### 3.1 工作原理

**立体匹配 (Stereo Matching):**
- 双目相机从不同视角获取同一场景的两幅图像
- 通过视差计算深度信息: `深度 = (基线 × 焦距) / 视差`
- 精度与基线距离、分辨率、环境光照相关

**结构光 (Structured Light):**
- 投射已知图案到物体表面
- 分析图案变形计算深度
- 如：Intel RealSense D400系列

**ToF (Time of Flight):**
- 发射调制光信号，测量返回时间
- 计算精确深度
- 如：Intel RealSense L515

### 3.2 常见双目深度相机

| 型号 | 分辨率 | 深度范围 | 接口 | 特点 |
|------|--------|----------|------|------|
| Intel RealSense D415 | 1280×720 | 0.5-10m | USB3.0 | 主动红外投影 |
| Intel RealSense D455 | 1280×720 | 0.4-20m | USB3.0/Type-C | 带IMU |
| Zed 2 | 2208×1242 | 0.2-20m | USB3.0 | 鱼眼镜头，高分辨率 |
| Zed 2i | 2208×1242 | 0.2-20m | USB3.0 | 工业级 |
| Orbbec Astra Pro | 640×480 | 0.6-8m | USB3.0 | 低成本 |

## 四、SCADA接入双目深度相机的可行性分析

### 4.1 直接接入方案 (❌ 不推荐)

**阻碍因素:**
1. **协议不兼容** — SCADA协议栈专为工业传感器设计，不支持视频流RTP/RTSP
2. **带宽限制** — 传统SCADA总线(RS485/Modbus)带宽仅几十kbps
3. **实时性冲突** — SCADA控制周期ms级，视频流GB级
4. **存储问题** — Historian数据库不适合视频归档

### 4.2 网关中转方案 (✅ 可行)

```
┌──────────────┐    OPC-UA/Modbus    ┌──────────────┐
│  双目深度相机 │ ──────────────► │   SCADA RTU   │
│   (视频流)   │    JSON/数据点    │   (网关)     │
└──────────────┘                   └──────────────┘
                                       │
                              ┌────────▼────────┐
                              │    SCADA Master │
                              │    (数据采集)  │
                              └─────────────────┘
```

**实现方式:**
- 部署边缘计算网关 (如NVIDIA Jetson、树莓派5)
- 网关提取关键数据点(深度值、目标位置)通过OPC-UA写入SCADA
- 视频流本地存储或上传到工业视频服务器

### 4.3 工业视频集成方案 (✅ 最佳)

```
┌──────────────┐                   ┌──────────────┐
│  双目深度相机 │ ─── RTSP/HTTP ──► │ 工业视频服务器│
│   (视频流)   │                   │ (ONVIF/RTSP) │
└──────────────┘                   └──────┬───────┘
                                           │
                                    ┌──────▼───────┐
                                    │  SCADA Master │
                                    │  (HMI集成)  │
                                    └──────────────┘
```

**标准协议:**
- **ONVIF** — 网络视频接口标准
- **RTSP** — 实时流协议
- **GB/T 28181** — 中国公共安全视频监控联网标准

## 五、推荐集成架构

### 5.1 轻量级方案 (小型SCADA)

```
深度相机 ──USB3.0──► 边缘网关(Jetson Nano) ──OPC-UA──► SCADA RTU ──Modbus──► SCADA Master
                         │
                    提取: 目标距离/坐标
                    存储: 视频流到本地SSD
```

### 5.2 工业级方案 (大型SCADA)

```
深度相机 ──RTSP──► 工业视频服务器 ──ONVIF/GB28181──► 监控网络 ──OPC-UA──► SCADA
                         │
                   ┌──────┴───────┐
                   │ 视频分析单元  │
                   │ (异常检测)   │
                   └──────────────┘
```

## 七、结论与建议

### ✅ SCADA可以接入双目深度相机，但需要中间层

**不可行:**
- SCADA直接采集视频流(协议/带宽不匹配)

**可行:**
- 通过边缘网关提取关键数据点
- 通过工业视频服务器集成视频流
- SCADA负责控制，数据单独处理

**推荐方案:**
1. Intel RealSense D455 + Jetson + OPC-UA网关
2. Zed 2i + 工业视频服务器 + ONVIF
